// ============================================================
[[hybrid-plugin-dev]]
== 14. Developing a Hybrid Plugin
// ============================================================

This section walks you through creating a new hybrid plugin from scratch — from
project setup to a running server with your mixin applied.

=== 14.1. Prerequisites

[cols="1,2"]
|===
|Prerequisite |Detail

|Java 21 JDK
|Required by Paper 1.21.11 and by Laminate's annotation processors.
`JAVA_HOME` must point at a JDK 21+ installation.

|Gradle 8+
|The `paperweight-userdev` plugin requires Gradle 8.
The Gradle wrapper (`./gradlew`) handles the version automatically.

|Laminate server JAR
|A running Laminate server to deploy to.
Build one with `./gradlew createLaminateServerJar` from the Laminate repository root.
|===

=== 14.2. Project structure

A minimal hybrid plugin project looks like this:

....
my-plugin/
├── build.gradle.kts
├── settings.gradle.kts
└── src/main/
    ├── java/com/example/myplugin/
    │   ├── MyPlugin.java
    │   └── mixins/
    │       └── ServerMixin.java
    └── resources/
        ├── plugin.yml
        └── mixins.myplugin.json
....

=== 14.3. `settings.gradle.kts`

If your plugin is developed as a standalone project (outside the Laminate monorepo):

[source,kotlin]
----
rootProject.name = "my-plugin"
----

If you are developing inside the Laminate monorepo alongside the server source, add an
optional include in the root `settings.gradle.kts` (the `optionalInclude` helper is already
present) and point `compileOnly(project(":laminate-api"))` at the monorepo project.

=== 14.4. `build.gradle.kts`

[source,kotlin]
----
plugins {
    java
    id("com.gradleup.shadow") version "9.4.1"           // <1>
    id("io.papermc.paperweight.userdev") version "2.0.0-beta.19"  // <2>
}

repositories {
    mavenCentral()
    maven("https://repo.papermc.io/repository/maven-public/")
    maven("https://maven.fabricmc.net/")   // sponge-mixin
}

dependencies {
    // Mixin API — compileOnly; provided at runtime by Laminate. Never shade.
    compileOnly("net.fabricmc:sponge-mixin:0.17.0+mixin.0.8.7")         // <3>
    compileOnly("io.github.llamalad7:mixinextras-common:0.5.3")          // <4>
    annotationProcessor("io.github.llamalad7:mixinextras-common:0.5.3")  // <5>

    // laminate-api: @GenerateAccessor, @InjectEvent, etc.
    compileOnly("eu.lschreiber.laminate:laminate-api:<version>")          // <6>
    annotationProcessor("eu.lschreiber.laminate:laminate-api:<version>")  // <7>

    // NMS access via paperweight userdev (Mojang-mapped NMS at compile time)
    paperweight.paperDevBundle("1.21.11-R0.1-SNAPSHOT")                   // <8>
}

java {
    toolchain.languageVersion.set(JavaLanguageVersion.of(21))
}

tasks {
    build { dependsOn(shadowJar) }

    processResources {
        filesMatching("plugin.yml") { expand("version" to version) }
    }
}
----
<1> Shadow is used to produce a fat JAR for deployment.
<2> `paperweight-userdev` provides Mojang-mapped NMS classes at compile time;
    without it you cannot reference NMS types in your mixin targets.
<3> SpongePowered Mixin annotations (`@Mixin`, `@Inject`, `@Unique`, etc.).
    `compileOnly` because Laminate puts this on the server classpath at runtime.
<4> MixinExtras annotations (`@WrapOperation`, etc.).
    Also `compileOnly` for the same reason.
<5> MixinExtras annotation processor generates helper code needed by MixinExtras
    at runtime.
<6> `laminate-api` — `compileOnly` because `paper-server` puts it on the server
    classpath via `implementation`.
<7> `laminate-api` annotation processor — generates accessor interfaces
    (`@GenerateAccessor`) and companion mixin classes (`@InjectEvent`).
<8> Provides Mojang-mapped NMS sources and classes so you can reference e.g.
    `net.minecraft.server.MinecraftServer` directly.

NOTE: When developing inside the Laminate monorepo, replace the Maven coordinate
`eu.lschreiber.laminate:laminate-api:<version>` with
`compileOnly(project(":laminate-api"))` and `annotationProcessor(project(":laminate-api"))`.

=== 14.5. `plugin.yml`

[source,yaml]
----
name: my-plugin
version: '$version'
main: com.example.myplugin.MyPlugin
api-version: '1.21'
authors: [ your-name ]

hybrid: true                          # <1>
mixins:
  - mixins.myplugin.json              # <2>
  - mixins.myplugin.generated.json    # <3>
----
<1> `hybrid: true` is the Laminate flag that triggers mixin bootstrap for this plugin.
    Omitting it makes Laminate treat the plugin as an ordinary Bukkit plugin.
<2> Your hand-written mixin configuration file (see <<mixin-config,§14.6>>).
<3> Generated by `@InjectEvent`; only needed when you use that annotation.
    If you do not use `@InjectEvent`, omit this line.

[[mixin-config]]
=== 14.6. Mixin configuration file (`mixins.myplugin.json`)

[source,json]
----
{
  "required": true,
  "package": "com.example.myplugin.mixins",    // <1>
  "compatibilityLevel": "JAVA_21",
  "mixins": [
    "ServerMixin"                               // <2>
  ]
}
----
<1> Must exactly match the Java package that contains your `@Mixin` classes.
<2> Simple class names (without the package prefix) of every mixin class.

=== 14.7. Writing a mixin class

[source,java]
----
package com.example.myplugin.mixins;

import net.minecraft.server.MinecraftServer;
import org.spongepowered.asm.mixin.Mixin;
import org.spongepowered.asm.mixin.Unique;

@Mixin(MinecraftServer.class)                 // <1>
public final class ServerMixin {

    @Unique                                    // <2>
    public void myPlugin$doSomething() {       // <3>
        System.out.println("Injected method called!");
    }
}
----
<1> `MinecraftServer.class` is available at compile time through `paperweight-userdev`.
    Use Mojang-mapped class names throughout; Laminate does not remap at runtime.
<2> `@Unique` marks this as a new method added by this mixin (not an override).
<3> Prefix every injected field and method name with `yourpluginid$` to avoid
    collisions with other plugins' mixins.  Use a unique prefix — collisions are a
    process-level error in SpongePowered Mixin.

=== 14.8. Generating an accessor interface with `@GenerateAccessor`

To call a mixin-injected method from your plugin's normal code, annotate it with
`@GenerateAccessor`:

[source,java]
----
import eu.lschreiber.laminateapi.accessor.GenerateAccessor;
import org.spongepowered.asm.mixin.Unique;

@Unique
@GenerateAccessor(name = "doSomething")        // <1>
public void myPlugin$doSomething() {
    System.out.println("Injected!");
}
----
<1> The processor generates `MinecraftServerAccessor` in the sibling `.accessor` package.

At compile time, the annotation processor creates:

[source,java]
----
// Generated — do not edit
package com.example.myplugin.accessor;

public interface MinecraftServerAccessor {

    void doSomething();

    static MinecraftServerAccessor of(Object target) {
        return LaminateAccessors.create(
            MinecraftServerAccessor.class, target,
            new String[]{"doSomething", "myPlugin$doSomething"});
    }
}
----

Using the accessor from your plugin code:

[source,java]
----
MinecraftServer nmsServer = ((CraftServer) Bukkit.getServer()).getServer();
MinecraftServerAccessor accessor = MinecraftServerAccessor.of(nmsServer);
accessor.doSomething();
----

=== 14.9. Firing Bukkit events from NMS code with `@InjectEvent`

`@InjectEvent` generates a companion mixin that calls `Bukkit.getPluginManager().callEvent(...)`
at the specified injection point, eliminating the need to write boilerplate `@Inject`
methods by hand.

==== 14.9.1. Define your event class

[source,java]
----
package com.example.myplugin.events;

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;
import net.minecraft.server.level.ServerLevel;

public class ServerTickEvent extends Event {

    private static final HandlerList HANDLERS = new HandlerList();
    private final ServerLevel level;

    public ServerTickEvent(ServerLevel level) {        // <1>
        this.level = level;
    }

    public ServerLevel getLevel() { return level; }
    @Override public HandlerList getHandlers() { return HANDLERS; }
    public static HandlerList getHandlerList() { return HANDLERS; }
}
----
<1> The first constructor parameter must be the NMS target object (`this` from the
    injected method); subsequent parameters match the extra arguments you declare on
    `@InjectEvent`.

==== 14.9.2. Annotate a marker method in your mixin

[source,java]
----
import eu.lschreiber.laminateapi.event.At;
import eu.lschreiber.laminateapi.event.InjectEvent;
import net.minecraft.server.level.ServerLevel;
import org.spongepowered.asm.mixin.Mixin;
import org.spongepowered.asm.mixin.Unique;
import java.util.function.BooleanSupplier;

@Mixin(ServerLevel.class)
public abstract class ServerLevelMixin {

    @Unique
    @InjectEvent(
        method = "tick",                                      // <1>
        at = At.HEAD,                                         // <2>
        event = ServerTickEvent.class                         // <3>
    )
    private void myPlugin$onTick(BooleanSupplier hasTime) {  // <4>
        // Marker — the annotation processor generates the real injection.
    }
}
----
<1> Simple name of the NMS method to inject into.  For overloaded methods, use
    `methodRewrite = @Rewrite(method = "tick", args = { BooleanSupplier.class })` instead.
<2> `At.HEAD` injects before the first instruction.  See `At` for other values.
<3> The event class to instantiate and fire.
<4> The method signature (excluding the receiver) becomes the extra arguments forwarded
    to the event constructor after `this`.

==== 14.9.3. Register the generated config in `plugin.yml`

The processor produces `mixins.myplugin.generated.json` alongside your compiled classes.
Add it to `plugin.yml`:

[source,yaml]
----
mixins:
  - mixins.myplugin.json
  - mixins.myplugin.generated.json
----

=== 14.10. Building and deploying

[source,bash]
----
# Build the fat JAR
./gradlew shadowJar

# The output is in:
# build/libs/my-plugin-<version>-all.jar
----

Copy `my-plugin-<version>-all.jar` to the Laminate server's `plugins/` directory.
Start (or restart) the server.
You will see Laminate log a line similar to:

----
[Laminate] Bootstrapped mixin for hybrid plugin: my-plugin
----

=== 14.11. Common mistakes

[cols="1,2"]
|===
|Mistake |Consequence and fix

|Shading `sponge-mixin`, `mixinextras-common`, or `laminate-api`
|Creates duplicate class definitions in the most sensitive class-loading layer.
Use `compileOnly` for all three; they are already on the server classpath.

|Missing `hybrid: true` in `plugin.yml`
|Laminate treats the plugin as an ordinary Bukkit plugin; mixins are never registered.

|Listing a mixin config in `plugin.yml` but forgetting to add the class to the JSON file
|`HybridDescriptorReader` verifies that every listed config entry exists as a JAR entry;
bootstrap fails and the plugin is removed from provider storage.

|Referencing remapped (Spigot/Obfuscated) class names in `@Mixin` targets
|`PaperQuiltLauncher.getTargetNamespace()` returns `"mojang"`.
Use Mojang-mapped names as provided by `paperweight-userdev`.

|Using `/reload` with an active hybrid plugin
|`ReloadCommand` blocks the command entirely when any `LaminatePluginProvider` is
registered.  Restart the server instead.

|Missing plugin-unique prefix on injected methods/fields
|SpongePowered Mixin raises a name collision error at bootstrap if two plugins inject
a method with the same name into the same target class.
|===
