This document describes the internal design and implementation of Laminate, a Paper 1.21.11 fork that adds first-class support for hybrid plugins: standard Bukkit/Paper plugins that may also register SpongePowered Mixin bytecode transformations against the live server without requiring operators to supply manual JVM flags or install a separate mod loader.

Every question about how the server starts, how mixins are registered, how the plugin provider system recognises hybrid plugins, and how laminate-api tools work for plugin authors should be answerable from the sections that follow.

1. How to read this document

Sections 2 through 6 cover the server-startup and mixin-bootstrap path in execution order: architecture overview, self-launching entry point, Java agent, bootstrap coordination, and the full mixin bootstrap subsystem.

Sections 7 and 8 cover the Quilt/Knot adapter (PaperQuiltLauncher) and the JVM class-file transformer (PaperMixinClassFileTransformer).

Section 9 covers the plugin provider system. Section 10 covers the laminate-api module (annotation processors and runtime bridges). Section 14 is a step-by-step guide to developing your own hybrid plugin.

All class references use their simple name where unambiguous; fully-qualified names are given the first time a class is introduced. Source paths are relative to the repository root.

2. Architecture Overview

2.1. What Laminate adds

Laminate does not replace Paper’s patch stack. It layers a narrow, deep capability on top of upstream Paper: loading plugins that can transform live server bytecode via SpongePowered Mixin.

The central problem is ordering. The JVM instrument API (ClassFileTransformer) must be active before a class is first loaded; certain Mixin operations (field addition) cannot be applied retroactively. Paper’s plugin system loads classes far too late. Laminate therefore pushes the entire mixin setup to the very beginning of the JVM’s life, before any Minecraft class is loaded.

2.4. Key design invariants

  • The ClassFileTransformer must be installed before any net.minecraft.* class is loaded. PaperBootstrap.boot() calls bootstrap before SharedConstants.tryDetectVersion().

  • Bootstrap is idempotent: a second call returns cached results immediately.

  • Failed hybrid plugins are isolated: their providers are removed from Paper’s provider storage; the rest of the server continues starting.

  • Mixin transformations are irreversible process state: hot-reload is blocked whenever any hybrid plugin is active.

3. Self-Launching Entry Point

Class

io.papermc.paper.LaminateLauncher

Source

paper-server/src/main/java/io/papermc/paper/LaminateLauncher.java

3.1. Purpose

The Laminate server JAR must be started with two things that ordinary operators do not supply:

  1. -javaagent:self — so PaperMixinAgent runs before main() and captures the JVM’s Instrumentation handle.

  2. Four --add-opens flags — so the Quilt/Knot mixin framework can access sealed JDK packages via reflection.

LaminateLauncher eliminates both requirements for the operator. It is the Main-Class in the distribution JAR’s manifest. On a first launch (no agent), it relaunches itself with all required flags. On the second JVM (agent already ran), it delegates to the Paperclip bundler.

3.2. Detection mechanism

PaperMixinAgent.premain() sets the system property laminate.agent.loaded before main() is called.

LaminateLauncher.main() checks for this property:

public static void main(final String[] args) throws Exception {
    if (System.getProperty(AGENT_LOADED_PROP) != null) {
        invokeBundlerMain(args);   // agent ran — delegate to Paperclip
        return;
    }
    relaunchWithAgent(args);       // first launch — relaunch with flags
}

AGENT_LOADED_PROP = "laminate.agent.loaded". The constant is shared between LaminateLauncher and PaperMixinAgent (the latter writes it; the former reads it).

3.3. Relaunch logic (relaunchWithAgent)

  1. Locates the Java executable via ProcessHandle.current().info().command(), falling back to $JAVA_HOME/bin/java.

  2. Locates the running JAR via LaminateLauncher.class.getProtectionDomain().getCodeSource().getLocation().

  3. Copies all existing JVM flags from ManagementFactory.getRuntimeMXBean().getInputArguments(), stripping any -javaagent: entry that points at the same JAR (to avoid a duplicate agent registration) while preserving all other agents (profilers, APM agents, security agents).

  4. Appends the four --add-opens flags required by Quilt/Knot:

    --add-opens=java.base/java.lang=ALL-UNNAMED
--add-opens=java.base/java.lang.reflect=ALL-UNNAMED
--add-opens=java.base/java.io=ALL-UNNAMED
--add-opens=java.base/java.util=ALL-UNNAMED

  5. Adds -javaagent:<selfJar>.

  6. Adds -jar <selfJar>.

  7. Forwards all application arguments unchanged (nogui, --port, --plugins, etc.).

  8. Starts the child process with ProcessBuilder.inheritIO() and exits with the child’s return code.

3.4. Bundler delegation (invokeBundlerMain)

Reads Bundler-Main-Class from the JAR manifest (set to io.papermc.paperclip.Main in createLaminateServerJar) and invokes its main(String[]) via reflection. Using the manifest attribute rather than a hard-coded class name keeps the launcher agnostic of the specific bundler implementation.

4. Java Agent

Class

io.papermc.paper.mixin.PaperMixinAgent

Source

paper-server/src/main/java/io/papermc/paper/mixin/PaperMixinAgent.java

4.1. Purpose

PaperMixinAgent is intentionally minimal. Its only job is to accept the JVM’s Instrumentation instance from the JVM launcher protocol and make it available to the rest of the bootstrap chain.

4.2. premain / agentmain

public static void premain(final String args, final Instrumentation inst) {
    System.setProperty(LAUNCHER_SIGNAL_PROP, "true");
    System.getProperties().put(INSTRUMENTATION_KEY, inst);
}

public static void agentmain(final String args, final Instrumentation inst) {
    premain(args, inst);
}

LAUNCHER_SIGNAL_PROP = "laminate.agent.loaded" — signals LaminateLauncher that this JVM is correctly configured.

INSTRUMENTATION_KEY = "paper.mixin.instrumentation" — the Instrumentation object is stored in System.getProperties() (a JVM-global Hashtable that is reachable across all class loaders) so that HybridPluginMixinBootstrap can retrieve it later without any shared static reference.

4.3. getInstrumentation()

static Instrumentation getInstrumentation() {
    final Object obj = System.getProperties().get(INSTRUMENTATION_KEY);
    return (obj instanceof Instrumentation inst) ? inst : null;
}

Returns null if the agent was never loaded; HybridPluginMixinBootstrap checks for this case and logs a clear warning rather than crashing.

5. Bootstrap Coordination

5.1. PaperBootstrap

Class

io.papermc.paper.PaperBootstrap

Source

paper-server/src/main/java/io/papermc/paper/PaperBootstrap.java

PaperBootstrap.boot(OptionSet) is the Paperclip-delegated entry point. The critical ordering is:

public static void boot(final OptionSet options) {
    bootstrapHybridMixins(options);          // (1) Laminate — before any Minecraft class

    SharedConstants.tryDetectVersion();      // (2) first Minecraft class load
    getStartupVersionMessages().forEach(LOGGER::info);
    Main.main(options);                      // (3) rest of Paper startup
}

SharedConstants.tryDetectVersion() triggers the class loading of core Minecraft classes. Any mixin transformation that adds fields to those classes must be applied before this line; once a class is loaded, field addition via retransformation is not guaranteed to succeed.

bootstrapHybridMixins(OptionSet) reads the --plugins, --add-plugin, and --add-plugin-dir options from the command-line and delegates to LaminateMixinBootstrap.bootstrap(pluginDir, addFiles, addDirs).

5.2. PluginInitializerManager

Class

io.papermc.paper.plugin.PluginInitializerManager

Source

paper-server/src/main/java/io/papermc/paper/plugin/PluginInitializerManager.java

PluginInitializerManager.load(OptionSet) is the second point where bootstrap is invoked (idempotency means the second call is essentially a no-op, returning the cached failure set from the first call). Its job at this stage is:

  1. Call LaminateMixinBootstrap.bootstrap(…​) with the same paths.

  2. Call EntrypointUtil.registerProvidersFromSource(…​) for the plugin directory and all --add-plugin/--add-plugin-dir inputs.

  3. Remove from all provider storages any provider whose lower-case name appears in failedHybridPlugins.

  4. Partition the remaining providers into three categories and log counts:

    Category Predicate

    Laminate plugins

    provider instanceof LaminatePluginProvider

    Paper plugins

    provider instanceof PaperPluginParent.PaperServerPluginProvider

    Bukkit/Legacy plugins

    provider instanceof SpigotPluginProvider (but not LaminatePluginProvider)

LaminatePluginProvider is checked first because it is a subtype of SpigotPluginProvider.

5.3. PluginInitializerManager.reload()

This method handles /reload. When any LaminatePluginProvider is registered, it emits a prominent warning:

You are reloading while having Laminate hybrid plugins installed.
Laminate hybrid plugins do NOT support being reloaded.
Mixin bytecode transformations are permanent for the lifetime of this server process.

This is a warning, not a hard block at the reload level; the hard block is in ReloadCommand, which refuses to run /reload at all when any hybrid plugin is active.

6. Mixin Bootstrap Subsystem

All classes in this section live under paper-server/src/main/java/io/papermc/paper/mixin/.

6.1. Public façade: LaminateMixinBootstrap

Class

io.papermc.paper.mixin.LaminateMixinBootstrap

public static Set<String> bootstrap(
    Path pluginDir,
    List<Path> addFiles,
    List<Path> addDirs
)

public static Set<String> getLoadedHybridPluginNames()

bootstrap delegates to HybridPluginMixinBootstrap.injectHybridMixins(…​) and returns the set of lower-case plugin names whose bootstrap failed (these are removed from provider storage by the caller).

getLoadedHybridPluginNames returns the names (original case) of plugins that were successfully bootstrapped, used by SpigotPluginProviderFactory to upgrade providers to LaminatePluginProvider.

6.2. JAR discovery: PluginJarScanner

Class

io.papermc.paper.mixin.PluginJarScanner

static Set<Path> collectPluginJars(
    Path defaultPluginDir,
    List<Path> additionalFiles,
    List<Path> additionalDirs
)

Collects all candidate plugin JARs into a de-duplicated HashSet<Path> of normalized, absolute paths.

Algorithm:

  1. Call collectFromDirectory(defaultPluginDir, jars).

  2. For each path in additionalFiles: add if it is an existing regular file ending in .jar.

  3. For each path in additionalDirs: call collectFromDirectory(dir, jars).

collectFromDirectory lists the directory with Files.list(), filters to regular .jar files, normalizes each path to absolute form, and adds it to the output set. A null or non-directory argument is silently ignored. An IOException during listing is logged as a warning; the partial results are kept.

6.3. Descriptor model: HybridDescriptor

record HybridDescriptor(String name, List<String> mixinConfigs) {}

A plain immutable record. name is the plugin name as declared in plugin.yml (original case). mixinConfigs is an unmodifiable list of mixin JSON resource paths (e.g. "mixins.myplugin.json").

6.4. Descriptor parsing: HybridDescriptorReader

Class

io.papermc.paper.mixin.HybridDescriptorReader

static HybridDescriptor read(Path pluginJar)
    throws IOException, InvalidConfigurationException

Opens the JAR with JarFile, then:

  1. Looks up the plugin.yml entry; returns null if absent (not a Bukkit plugin).

  2. Reads and parses the YAML with YamlConfiguration.loadFromString(…​).

  3. Checks hybrid: true; returns null if false or absent.

  4. Reads name; throws InvalidConfigurationException if blank or absent.

  5. Reads mixins as List<String>; throws if empty.

  6. For each config name, verifies a JarEntry with that name exists in the JAR; throws if missing.

  7. Returns new HybridDescriptor(name, Collections.unmodifiableList(mixinConfigs)).

6.5.2. injectHybridMixins — main entry point

public static Set<String> injectHybridMixins(
    Path defaultPluginDir,
    List<Path> addPluginFiles,
    List<Path> addPluginDirs
)

  1. Returns cachedFailedPlugins immediately if non-null (idempotency).

  2. Calls PluginJarScanner.collectPluginJars(…​).

  3. For each JAR: calls processPluginJar(pluginJar, failedPlugins).

  4. If any injection succeeded or Mixin is already bootstrapped, and phases have not yet been advanced: calls finishMixinPhases(), then installClassFileTransformer().

  5. If any plugins failed: logs a warning with the count.

  6. Caches and returns Set.copyOf(failedPlugins).

6.5.3. processPluginJar

  1. Calls HybridDescriptorReader.read(pluginJar).

  2. If null: returns false (not a hybrid plugin).

  3. Calls injectDescriptor(pluginJar, descriptor).

  4. On any Throwable: logs a warning with the JAR name/plugin name and adds the lower-case plugin name to failedPlugins; returns false.

  5. Returns true on success.

6.5.4. injectDescriptor

  1. Calls ensureMixinBootstrap() (creates PaperQuiltLauncher, sets Mixin service, calls MixinBootstrap.init() and MixinExtrasBootstrap.init()).

  2. Creates a URLClassLoader over the plugin JAR URL, with the thread-context class loader as parent.

  3. Adds the loader to PINNED_MIXIN_LOADERS (keeps it from being GC’d).

  4. Calls registerPluginClassLoader(mixinResourceLoader) to register it with the launcher.

  5. Calls registerMixinConfigs(mixinResourceLoader, descriptor).

  6. Adds the plugin name to LOADED_HYBRID_PLUGIN_NAMES.

6.5.5. registerMixinConfigs

private static void registerMixinConfigs(
    URLClassLoader loader,
    HybridDescriptor descriptor
)

Temporarily switches Thread.currentThread().getContextClassLoader() to loader so the Mixin subsystem’s ClassLoader.getResourceAsStream(config) calls find the mixin JSON files inside the plugin JAR. Then calls Mixins.addConfiguration(config) for each entry in descriptor.mixinConfigs(). Restores the previous context class loader in finally.

6.5.6. ensureMixinBootstrap (synchronized)

private static synchronized void ensureMixinBootstrap()

One-time initialisation:

  1. Creates launcher = new PaperQuiltLauncher().

  2. Sets System.setProperty("mixin.service", MixinServiceKnot.class.getName()).

  3. Calls MixinBootstrap.init().

  4. Calls MixinExtrasBootstrap.init().

MixinBootstrap.init() expects the Quilt launcher to be registered; the PaperQuiltLauncher constructor calls QuiltLauncherBase.setLauncher(this) for this reason.

6.5.7. finishMixinPhases

MixinEnvironment.gotoPhase(Phase) is package-private. Laminate accesses it via reflection (the same technique used by Quilt’s own QuiltLauncherBase):

Method gotoPhase = MixinEnvironment.class.getDeclaredMethod(
    "gotoPhase", MixinEnvironment.Phase.class);
gotoPhase.setAccessible(true);
gotoPhase.invoke(null, MixinEnvironment.Phase.INIT);
gotoPhase.invoke(null, MixinEnvironment.Phase.DEFAULT);

This advances the environment from PREINIT → INIT → DEFAULT. After this point, mixin configurations are locked; no more configs can be added.

6.5.8. installClassFileTransformer

  1. Obtains the IMixinTransformer from MixinServiceKnot.getTransformer() via reflection.

  2. Retrieves Instrumentation from PaperMixinAgent.getInstrumentation().

  3. If null: logs a warning that transformations will not apply and returns.

  4. Creates new PaperMixinClassFileTransformer(mixinTransformer).

  5. Calls instrumentation.addTransformer(transformer, true) (true = supports retransformation).

  6. Calls retransformAlreadyLoadedClasses(instrumentation) as a safety net.

6.5.9. retransformAlreadyLoadedClasses

Finds all classes loaded before the transformer was installed that start with net.minecraft. and are marked modifiable by instrumentation.isModifiableClass(). Retransforms each individually with instrumentation.retransformClasses(clazz).

Individual retransformation (one class per call) is intentional: a UnsupportedOperationException on one class (schema-changing mixin against an already- loaded class) can be caught and logged without aborting the rest.

7. Quilt/Knot Adapter

Class

io.papermc.paper.mixin.PaperQuiltLauncher

Source

paper-server/src/main/java/io/papermc/paper/mixin/PaperQuiltLauncher.java

7.1. Purpose

MixinServiceKnot (the Mixin service implementation shipped by Quilt Loader) expects to find a QuiltLauncher registered in QuiltLauncherBase from which it can:

  • look up class bytes

  • look up resources

  • query whether a class is already loaded

  • retrieve mapping configuration

PaperQuiltLauncher satisfies this contract, bridging Paper’s runtime environment into the Quilt/Knot mixin infrastructure.

7.2. Key design decisions vs. Quilt’s Knot

Feature Quilt Knot PaperQuiltLauncher

Class bytes

Read via KnotClassDelegate from Knot’s own class loader

Read from server class loader, then thread-context loader, then registered plugin loaders

isClassLoaded

Probes findLoadedClass() honestly

Always returns false to prevent MixinTargetAlreadyLoadedException; already-loaded classes are handled by retransformation

Mapping configuration

Full Quilt mapping pipeline

Returns null; Paper ships Mojang-mapped and no runtime remapping is needed

Target namespace

Configured at launch

Hard-coded to "mojang"

Class-path management

Knot dynamically adds paths to its loader

Tracks registered URLClassLoader instances in a CopyOnWriteArrayList

Game transformer

GameTransformer with entry-point processing

Returns an empty no-op GameTransformer

7.3. Resource search order

Both findResource(String name) and findResourceUrl(String name) follow the same three-step search:

  1. Server class loader (getClass().getClassLoader()).

  2. Thread-context class loader, if it differs from the server loader.

  3. Each registered plugin URLClassLoader in insertion order.

Returns the first result found, or null.

Thread safety: pluginClassLoaders is a CopyOnWriteArrayList. Writes (one per hybrid plugin during bootstrap) are rare; reads (on every class load from the Mixin transformer) are frequent. CopyOnWriteArrayList provides lock-free snapshot iteration for reads.

7.4. No-op overrides

Several QuiltLauncher methods are relevant only to Quilt’s own class-isolation model and are intentionally empty or return fixed values in PaperQuiltLauncher:

  • setAllowedPrefixes — Paper does not enforce class-prefix isolation.

  • setTransformCache — Paper does not use Quilt’s transform caching.

  • setHiddenClasses — Paper does not enforce class hiding.

  • setPluginPackages — Paper does not enforce plugin-package isolation.

  • hideParentUrl / hideParentPath — not applicable.

  • validateGameClassLoader — not applicable.

  • getManifest — returns null.

8. JVM Class-File Transformer

Class

io.papermc.paper.mixin.PaperMixinClassFileTransformer

Source

paper-server/src/main/java/io/papermc/paper/mixin/PaperMixinClassFileTransformer.java

8.1. Lifecycle

  1. Created by HybridPluginMixinBootstrap.installClassFileTransformer().

  2. Registered with the JVM via Instrumentation.addTransformer(transformer, true).

  3. For every class the JVM loads or retransforms, the JVM calls transform(…​).

8.2. transform implementation

public byte[] transform(
    ClassLoader loader,
    String className,        // slash-separated, e.g. "net/minecraft/server/MinecraftServer"
    Class<?> classBeingRedefined,
    ProtectionDomain protectionDomain,
    byte[] classfileBuffer
)

  1. Returns null immediately if className == null.

  2. Converts className from slash-separated to dot-separated.

  3. Calls canTransform(dotName); returns null if blocked.

  4. Delegates to mixinTransformer.transformClassBytes(dotName, dotName, classfileBuffer).

  5. Returns the transformed bytes if they differ from the input; returns null otherwise.

  6. Catches any Throwable: logs an error and returns null (class loads untransformed).

Returning null from a ClassFileTransformer signals "no change" to the JVM; this is correct and avoids unnecessary byte-array copies for classes that no mixin targets.

8.3. canTransform — exclusion list

The following prefixes are excluded from transformation:

Excluded prefix Reason

org.apache.logging.log4j.

Logging infrastructure; transforming it would cause circularity during error logging inside the transformer.

org.spongepowered.asm.

Mixin framework itself; would be a circular dependency.

org.objectweb.asm.

ASM bytecode library used by Mixin; same reason.

io.papermc.paper.mixin.

Laminate’s own mixin infrastructure; transforming it could break the transformer itself.

10.3. @GenerateAccessor and GenerateAccessorProcessor

10.3.1. Purpose

Mixin classes inject new methods into server classes. Plugin code cannot call those methods directly (the plugin classloader has no compile-time reference to the injected method on the target class). @GenerateAccessor automates generating a typed Java interface through which the mixin-injected method can be called.

10.3.2. @GenerateAccessor annotation

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface GenerateAccessor {
    String name() default "";  // accessor method name; defaults to the annotated method's name
}

Applied to a @Unique method inside a @Mixin class. @Retention(RUNTIME) is required because LaminateAccessors reads it at runtime to map proxy calls back to the mixin method name.

10.3.3. GenerateAccessorProcessor — processing rounds

@SupportedAnnotationTypes("eu.lschreiber.laminateapi.accessor.GenerateAccessor") @SupportedSourceVersion(SourceVersion.RELEASE_21)

Each annotation-processing round:

  1. Iterates all @GenerateAccessor-annotated elements, keeping only ExecutableElement instances (method elements).

  2. For each method:

    1. Gets the enclosing TypeElement (the mixin class).

    2. Resolves the accessor method name: annotation.name() if non-blank and a valid Java identifier (validated with SourceVersion.isIdentifier), otherwise the method’s own simple name.

    3. Collects parameters with erased types and positional names p0, p1, … (positional names ensure the generated interface compiles even without -parameters).

    4. Groups entries by mixin class.

    5. Calls MixinTargetResolver.resolve(mixinClass, env, "@GenerateAccessor") to find the @Mixin target.

  3. For each mixin class with at least one entry: calls writeAccessorFile(…​).

10.3.4. Accessor package derivation

static String deriveAccessorPackage(final String mixinPackage) {
    if (mixinPackage.isEmpty()) return "accessor";
    final int lastDot = mixinPackage.lastIndexOf('.');
    if (lastDot < 0) return "accessor";
    return mixinPackage.substring(0, lastDot) + ".accessor";
}

Example: eu.lschreiber.testplugin.mixins → eu.lschreiber.testplugin.accessor.

The accessor is placed in a sibling .accessor package, not inside the mixin package. SpongePowered Mixin does not allow classes in the mixin package itself to be loaded directly by the application class loader; placing accessors outside the mixin package avoids this restriction.

10.3.5. Generated interface structure

For a mixin class targeting MinecraftServer with one @GenerateAccessor method, the processor generates MinecraftServerAccessor.java:

package eu.lschreiber.testplugin.accessor;

import eu.lschreiber.laminateapi.accessor.LaminateAccessors;

/**
 * Generated by {@link eu.lschreiber.laminateapi.accessor.GenerateAccessorProcessor}.
 * Do not edit manually.
 */
public interface MinecraftServerAccessor {

    void write(String p0);           // one method per @GenerateAccessor

    static MinecraftServerAccessor of(Object target) {
        return LaminateAccessors.create(
            MinecraftServerAccessor.class, target,
            new String[]{"write", "testPlugin$write"});
    }
}

The of(Object) factory encodes a flat mapping array [accessorName, mixinMethodName, …​] for LaminateAccessors.create.

10.4. LaminateAccessors

Class

eu.lschreiber.laminateapi.accessor.LaminateAccessors

LaminateAccessors is the runtime half of the accessor system. Generated of(Object) factories call LaminateAccessors.create(…​).

10.4.1. create

public static <T> T create(
    Class<T> accessorInterface,
    Object target,
    String[] methodMappings    // flat [accessorName, mixinName, ...]
)

  1. Validates that methodMappings.length is even.

  2. Builds a Map<String,String> from the flat array (accessor name → mixin method name).

  3. Retrieves or creates a ConcurrentHashMap<String,MethodHandle> keyed by target.getClass() from a process-global ConcurrentHashMap cache.

  4. For each interface method: looks up the mixin method name, builds a descriptor key "mixinName(paramType1,paramType2)", resolves and caches the MethodHandle if absent.

  5. Returns Proxy.newProxyInstance(…​) with an AccessorInvocationHandler.

10.4.2. MethodHandle resolution

private static MethodHandles.Lookup obtainLookup(Class<?> targetClass) {
    try {
        return MethodHandles.privateLookupIn(targetClass, MethodHandles.lookup());
    } catch (IllegalAccessException e) {
        return MethodHandles.lookup();
    }
}

Uses MethodHandles.privateLookupIn for wide access; falls back to the public lookup. The resolved handle is cached to avoid repeated reflection on every call.

Resolution walks the full class hierarchy (cls = cls.getSuperclass() until null), matching by name and erased parameter types.

10.4.3. AccessorInvocationHandler

A private record implementing InvocationHandler. Routes calls to Object methods (toString, hashCode, equals) to safe defaults, and all other calls through the cached MethodHandle:

return handle.invokeWithArguments(prependTarget(target, args));

prependTarget prepends the mixin target object as the first argument, because unreflected method handles for instance methods take the receiver as argument 0.

10.5. @InjectEvent, At, Rewrite, and InjectEventProcessor

10.5.1. Purpose

Plugin authors often want to fire a Bukkit event from inside an NMS method body. The naive approach (writing a companion mixin by hand) is repetitive and error-prone. @InjectEvent expresses the intent declaratively; the processor generates the companion mixin automatically.

10.5.2. At — injection point

Value Meaning

HEAD

Before the first instruction in the target method.

TAIL

Just before every RETURN in the target method.

RETURN

Immediately before each RETURN opcode (same as TAIL in most cases).

INVOKE

Before a specific method call; requires @InjectEvent#target() to name the call site.

Maps directly to SpongePowered Mixin’s @At value.

10.5.3. @Rewrite — type-safe method descriptor

@Target({})
@Retention(RetentionPolicy.CLASS)
public @interface Rewrite {
    String method();
    Class<?> returns() default void.class;
    Class<?>[] args() default {};
}

Used as the value of InjectEvent.methodRewrite() to construct an unambiguous JVM method descriptor for overloaded NMS methods. InjectEventProcessor converts a @Rewrite into a JVM descriptor string using JvmDescriptors.fromClass(…​).

10.5.4. @InjectEvent annotation

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.CLASS)
public @interface InjectEvent {
    String method() default "";           // simple name or full JVM descriptor
    Rewrite methodRewrite() default @Rewrite(method = "");  // type-safe alternative
    At at();
    Class<?> event();
    boolean cancellable() default false;
    String target() default "";           // for At.INVOKE: call-site descriptor
}

Exactly one of method and methodRewrite must be set; setting both or neither is a compile-time error emitted by InjectEventProcessor.

@Retention(CLASS) — not needed at runtime; the annotation is consumed entirely at compile time and only the generated companion mixin class remains.

10.5.5. InjectEventProcessor — generated artefacts

For each @InjectEvent-annotated method the processor generates:

  1. A companion mixin class (e.g. Generated_testPlugin$onTick_Mixin) annotated with @Mixin targeting the same NMS class.

  2. Inside the companion: a @Inject method with the appropriate @At and target descriptor.

  3. The injected body calls LaminateEventBridge.fire(…​) or LaminateEventBridge.fireAndReturn(…​) (when cancellable = true) with the fully-qualified event class name, the NMS target object, and any extra arguments.

All generated companion classes for a plugin are collected into a single mixin JSON configuration file named mixins.<plugin>.generated.json.

The plugin name is derived from the mixin class’s package. For eu.lschreiber.testplugin.mixins.ServerLevelMixin, the derived plugin name is testplugin, producing mixins.testplugin.generated.json.

That file must be listed in plugin.yml:

mixins:
  - mixins.testplugin.json
  - mixins.testplugin.generated.json

10.5.6. Cancellable injection — generated pattern

When cancellable = true, the generated @Inject method signature includes a CallbackInfo parameter, and the generated body becomes:

Event event = LaminateEventBridge.fireAndReturn(
    "eu.lschreiber.testplugin.events.ServerLevelTickStartEvent",
    (Object) this,
    hasTime
);
if (event instanceof Cancellable c && c.isCancelled()) {
    ci.cancel();
}

10.6. LaminateEventBridge

Class

eu.lschreiber.laminateapi.event.LaminateEventBridge

10.6.1. The class-loader problem

Generated companion mixin classes are injected into NMS classes (e.g. ServerLevel). NMS classes are loaded by the server’s AppClassLoader. Plugin event classes (e.g. ServerLevelTickStartEvent) are loaded by child PluginClassLoader instances. AppClassLoader cannot see classes defined in child class loaders.

A direct new ServerLevelTickStartEvent(…​) in the generated bytecode would cause a NoClassDefFoundError at runtime.

LaminateEventBridge solves this by being on AppClassLoader’s classpath (as part of `laminate-api, an implementation dependency of paper-server) while performing dynamic class resolution through plugin class loaders at runtime.

10.6.2. fire and fireAndReturn

public static void fire(String eventClassName, Object target, Object... extraArgs)

public static Event fireAndReturn(String eventClassName, Object target, Object... extraArgs)

Both delegate to constructAndFire(…​) which:

  1. Calls resolveConstructor(eventClassName, target, extraArgs).

  2. If the result is UNRESOLVED_SENTINEL: returns null.

  3. Prepends target to extraArgs to build the constructor argument array.

  4. Instantiates the event class with ctor.newInstance(ctorArgs).

  5. Calls Bukkit.getPluginManager().callEvent(event).

  6. Returns the event (or null on Exception).

10.6.3. Constructor resolution and caching

Cache key: "<eventFQN>|<targetClass>,<argType1>,<argType2>,…​"

On cache miss:

  1. Attempts to load the event class from every registered plugin’s class loader via plugin.getClass().getClassLoader().loadClass(eventClassName).

  2. If no plugin can load it: logs a warning, stores UNRESOLVED_SENTINEL, returns.

  3. Iterates eventClass.getDeclaredConstructors() to find one whose parameter types are compatible with the runtime argument types.

  4. Compatibility is checked per-parameter: null matches any non-primitive; boxed types match their primitives; standard isAssignableFrom applies for the rest.

  5. Calls ctor.setAccessible(true) and stores the result with putIfAbsent.

UNRESOLVED_SENTINEL is Object.class.getDeclaredConstructor() — a Constructor<Object> instance used purely as a sentinel to avoid repeated failed lookups.

10.7. Internal utilities

10.7.1. MixinTargetResolver

Class

eu.lschreiber.laminateapi.internal.MixinTargetResolver

Shared by both GenerateAccessorProcessor and InjectEventProcessor. Resolves the single target class declared in a @Mixin annotation on a given TypeElement.

Supports two forms:

  • @Mixin(SomeClass.class) — reads the value attribute as List<AnnotationValue>, extracts the first TypeMirror, resolves it to a TypeElement.

  • @Mixin(targets = "com.example.SomeClass") — reads the targets attribute as List<AnnotationValue>, extracts the first string, splits on the last . to get simple name.

Emits a compile-time error if:

  • More than one target is declared (multiple-target mixins are not supported by either processor).

  • No @Mixin annotation is found on the enclosing class.

Returns a MixinTarget(String simpleName, String fqn) record, or null on failure.

12.3. createLaminateServerJar task

Produces laminate-server-<version>.jar — the single-file operator distribution.

val createLaminateServerJar by tasks.registering(Jar::class) {
    archiveBaseName.set("laminate-server")
    duplicatesStrategy = DuplicatesStrategy.EXCLUDE

    // 1. New/updated classes first (shadow anything in the bundler JAR)
    from(sourceSets.main.get().output) {
        include(
            "io/papermc/paper/LaminateLauncher.class",
            "io/papermc/paper/mixin/PaperMixinAgent.class",
        )
    }

    // 2. Merge the entire mojmap bundler JAR
    from(tasks.createMojmapBundlerJar.flatMap { it.outputZip }.map {
        archiveOps.zipTree(it.asFile)
    }) {
        exclude("META-INF/MANIFEST.MF")
        exclude("META-INF/*.SF", "META-INF/*.RSA", "META-INF/*.DSA")
    }

    manifest {
        attributes(
            "Main-Class"              to "io.papermc.paper.LaminateLauncher",
            "Bundler-Main-Class"      to "io.papermc.paperclip.Main",
            "Premain-Class"           to "io.papermc.paper.mixin.PaperMixinAgent",

            "Can-Retransform-Classes" to "true",

}

13. Operator and Author Guidance

13.1. Minimum requirements

Requirement Detail

Java version

Java 21 or later (required by Paper 1.21.11 and by Laminate’s annotation processors).

Launch command

java -jar laminate-server-<version>.jar [server-args] — no additional JVM flags needed.

Plugin directory

Standard Paper plugins/ directory; hybrid plugins are discovered alongside ordinary plugins.

13.2. How Laminate selects JVM flags on relaunch

The --add-opens flags are required because Quilt Loader (and by extension MixinServiceKnot) uses reflection to access sealed JDK packages:

Flag Why needed

--add-opens=java.base/java.lang=ALL-UNNAMED

Accessing ClassLoader internals for class bytes.

--add-opens=java.base/java.lang.reflect=ALL-UNNAMED

Accessing method and field reflective objects.

--add-opens=java.base/java.io=ALL-UNNAMED

Accessing InputStream internals for resource loading.

--add-opens=java.base/java.util=ALL-UNNAMED

Accessing Hashtable (used by System.getProperties()).

13.3. /reload behaviour

Laminate blocks /reload at the command level when any LaminatePluginProvider is active. The underlying reason: ClassFileTransformer modifications are committed to the JVM’s class bytes for the lifetime of the process. There is no JVM API to undo a bytecode transformation once applied.

13.4. Writing a hybrid plugin — checklist

  1. Declare hybrid: true in plugin.yml.

  2. List every mixin JSON config file in the mixins: array. If using @InjectEvent, include mixins.<plugin>.generated.json.

  3. Target all mixins against Mojang-mapped names. PaperQuiltLauncher.getTargetNamespace() returns "mojang"; no runtime remapping is provided.

  4. Prefix all mixin-injected fields and methods with a plugin-unique namespace (yourpluginid$). Laminate does not change Mixin’s collision model.

  5. Do not shade laminate-api, sponge-mixin, or mixinextras-common. These are on the server classpath.

  6. Restart (not reload) is the lifecycle boundary for hybrid plugins.

13.5. Failed hybrid bootstrap

If HybridDescriptorReader.read() throws, or any step of injectDescriptor fails, the plugin’s lower-case name is added to failedHybridPlugins. After scanning all JARs, PluginInitializerManager.load() removes any provider whose name matches a failed plugin. The server continues starting; the broken plugin is simply absent. Logs will contain a warning identifying the failure with the JAR path and exception.

  - mixins.myplugin.json              # (2)
  - mixins.myplugin.generated.json    # (3)

  1. hybrid: true is the Laminate flag that triggers mixin bootstrap for this plugin. Omitting it makes Laminate treat the plugin as an ordinary Bukkit plugin.

  2. Your hand-written mixin configuration file (see §14.6).

  3. Generated by @InjectEvent; only needed when you use that annotation. If you do not use @InjectEvent, omit this line.

14.6. Mixin configuration file (mixins.myplugin.json)

{
  "required": true,
  "package": "com.example.myplugin.mixins",    // (1)
  "compatibilityLevel": "JAVA_21",
  "mixins": [
    "ServerMixin"                               // (2)
  ]
}

  1. Must exactly match the Java package that contains your @Mixin classes.

  2. Simple class names (without the package prefix) of every mixin class.

14.7. Writing a mixin class

package com.example.myplugin.mixins;

import net.minecraft.server.MinecraftServer;
import org.spongepowered.asm.mixin.Mixin;
import org.spongepowered.asm.mixin.Unique;

@Mixin(MinecraftServer.class)                 // (1)
public final class ServerMixin {

    @Unique                                    // (2)
    public void myPlugin$doSomething() {       // (3)
        System.out.println("Injected method called!");
    }
}

  1. MinecraftServer.class is available at compile time through paperweight-userdev. Use Mojang-mapped class names throughout; Laminate does not remap at runtime.

  2. @Unique marks this as a new method added by this mixin (not an override).

  3. Prefix every injected field and method name with yourpluginid$ to avoid collisions with other plugins' mixins. Use a unique prefix — collisions are a process-level error in SpongePowered Mixin.

14.8. Generating an accessor interface with @GenerateAccessor

To call a mixin-injected method from your plugin’s normal code, annotate it with @GenerateAccessor:

import eu.lschreiber.laminateapi.accessor.GenerateAccessor;
import org.spongepowered.asm.mixin.Unique;

@Unique
@GenerateAccessor(name = "doSomething")        // (1)
public void myPlugin$doSomething() {
    System.out.println("Injected!");
}

  1. The processor generates MinecraftServerAccessor in the sibling .accessor package.

At compile time, the annotation processor creates:

// Generated — do not edit
package com.example.myplugin.accessor;

public interface MinecraftServerAccessor {

    void doSomething();

    static MinecraftServerAccessor of(Object target) {
        return LaminateAccessors.create(
            MinecraftServerAccessor.class, target,
            new String[]{"doSomething", "myPlugin$doSomething"});
    }
}

Using the accessor from your plugin code:

MinecraftServer nmsServer = ((CraftServer) Bukkit.getServer()).getServer();
MinecraftServerAccessor accessor = MinecraftServerAccessor.of(nmsServer);
accessor.doSomething();

14.9. Firing Bukkit events from NMS code with @InjectEvent

@InjectEvent generates a companion mixin that calls Bukkit.getPluginManager().callEvent(…​) at the specified injection point, eliminating the need to write boilerplate @Inject methods by hand.

14.9.1. Define your event class

package com.example.myplugin.events;

import org.bukkit.event.Event;
import org.bukkit.event.HandlerList;
import net.minecraft.server.level.ServerLevel;

public class ServerTickEvent extends Event {

    private static final HandlerList HANDLERS = new HandlerList();
    private final ServerLevel level;

    public ServerTickEvent(ServerLevel level) {        // (1)
        this.level = level;
    }

    public ServerLevel getLevel() { return level; }
    @Override public HandlerList getHandlers() { return HANDLERS; }
    public static HandlerList getHandlerList() { return HANDLERS; }
}

  1. The first constructor parameter must be the NMS target object (this from the injected method); subsequent parameters match the extra arguments you declare on @InjectEvent.

14.9.2. Annotate a marker method in your mixin

import eu.lschreiber.laminateapi.event.At;
import eu.lschreiber.laminateapi.event.InjectEvent;
import net.minecraft.server.level.ServerLevel;
import org.spongepowered.asm.mixin.Mixin;
import org.spongepowered.asm.mixin.Unique;
import java.util.function.BooleanSupplier;

@Mixin(ServerLevel.class)
public abstract class ServerLevelMixin {

    @Unique
    @InjectEvent(
        method = "tick",                                      // (1)
        at = At.HEAD,                                         // (2)
        event = ServerTickEvent.class                         // (3)
    )
    private void myPlugin$onTick(BooleanSupplier hasTime) {  // (4)
        // Marker — the annotation processor generates the real injection.
    }
}

  1. Simple name of the NMS method to inject into. For overloaded methods, use methodRewrite = @Rewrite(method = "tick", args = { BooleanSupplier.class }) instead.

  2. At.HEAD injects before the first instruction. See At for other values.

  3. The event class to instantiate and fire.

  4. The method signature (excluding the receiver) becomes the extra arguments forwarded to the event constructor after this.

14.9.3. Register the generated config in plugin.yml

The processor produces mixins.myplugin.generated.json alongside your compiled classes. Add it to plugin.yml:

mixins:
  - mixins.myplugin.json
  - mixins.myplugin.generated.json

14.10. Building and deploying

# Build the fat JAR
./gradlew shadowJar

# The output is in:
# build/libs/my-plugin-<version>-all.jar

Copy my-plugin-<version>-all.jar to the Laminate server’s plugins/ directory. Start (or restart) the server. You will see Laminate log a line similar to:

[Laminate] Bootstrapped mixin for hybrid plugin: my-plugin

14.11. Common mistakes

Mistake Consequence and fix

Shading sponge-mixin, mixinextras-common, or laminate-api

Creates duplicate class definitions in the most sensitive class-loading layer. Use compileOnly for all three; they are already on the server classpath.

Missing hybrid: true in plugin.yml

Laminate treats the plugin as an ordinary Bukkit plugin; mixins are never registered.

Listing a mixin config in plugin.yml but forgetting to add the class to the JSON file

HybridDescriptorReader verifies that every listed config entry exists as a JAR entry; bootstrap fails and the plugin is removed from provider storage.

Referencing remapped (Spigot/Obfuscated) class names in @Mixin targets

PaperQuiltLauncher.getTargetNamespace() returns "mojang". Use Mojang-mapped names as provided by paperweight-userdev.

Using /reload with an active hybrid plugin

ReloadCommand blocks the command entirely when any LaminatePluginProvider is registered. Restart the server instead.

Missing plugin-unique prefix on injected methods/fields

SpongePowered Mixin raises a name collision error at bootstrap if two plugins inject a method with the same name into the same target class.
