// ============================================================
[[laminate-api]]
== 10. `laminate-api`
// ============================================================

`laminate-api` is a distinct Gradle module (`laminate-api/`) that combines:

* Annotation processors (run at plugin compile time).
* Runtime support classes (placed on the server's classpath by `paper-server`).

=== 10.1. Intended dependency model

For plugin authors:

[source,kotlin]
----
dependencies {
    compileOnly(project(":laminate-api"))
    annotationProcessor(project(":laminate-api"))
}
----

At runtime, `laminate-api` is an `implementation` dependency of `paper-server` and is
therefore on the server `AppClassLoader`'s classpath.
Plugins must *not* shade `laminate-api`, `sponge-mixin`, or `mixinextras-common`; the
server already provides them, and shading would create competing class definitions in the
most sensitive part of the mixin path.

=== 10.2. Package layout

[cols="1,2"]
|===
|Package |Contents

|`eu.lschreiber.laminateapi.accessor`
|`@GenerateAccessor`, `GenerateAccessorProcessor`, `LaminateAccessors`

|`eu.lschreiber.laminateapi.event`
|`@InjectEvent`, `InjectEventProcessor`, `LaminateEventBridge`, `At`, `Rewrite`,
`RewriteProcessor`

|`eu.lschreiber.laminateapi.internal`
|`MixinTargetResolver`, `JvmDescriptors` (not part of public API)
|===

[[generate-accessor]]
=== 10.3. `@GenerateAccessor` and `GenerateAccessorProcessor`
// ============================================================

==== 10.3.1. Purpose

Mixin classes inject new methods into server classes.
Plugin code cannot call those methods directly (the plugin classloader has no compile-time
reference to the injected method on the target class).
`@GenerateAccessor` automates generating a typed Java interface through which the
mixin-injected method can be called.

==== 10.3.2. `@GenerateAccessor` annotation

[source,java]
----
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface GenerateAccessor {
    String name() default "";  // accessor method name; defaults to the annotated method's name
}
----

Applied to a `@Unique` method inside a `@Mixin` class.
`@Retention(RUNTIME)` is required because `LaminateAccessors` reads it at runtime to map
proxy calls back to the mixin method name.

==== 10.3.3. `GenerateAccessorProcessor` — processing rounds

`@SupportedAnnotationTypes("eu.lschreiber.laminateapi.accessor.GenerateAccessor")`
`@SupportedSourceVersion(SourceVersion.RELEASE_21)`

Each annotation-processing round:

. Iterates all `@GenerateAccessor`-annotated elements, keeping only `ExecutableElement`
  instances (method elements).
. For each method:
  .. Gets the enclosing `TypeElement` (the mixin class).
  .. Resolves the accessor method name: `annotation.name()` if non-blank and a valid
     Java identifier (validated with `SourceVersion.isIdentifier`), otherwise the
     method's own simple name.
  .. Collects parameters with erased types and positional names `p0`, `p1`, … (positional
     names ensure the generated interface compiles even without `-parameters`).
  .. Groups entries by mixin class.
  .. Calls `MixinTargetResolver.resolve(mixinClass, env, "@GenerateAccessor")` to find
     the `@Mixin` target.
. For each mixin class with at least one entry: calls `writeAccessorFile(...)`.

==== 10.3.4. Accessor package derivation

[source,java]
----
static String deriveAccessorPackage(final String mixinPackage) {
    if (mixinPackage.isEmpty()) return "accessor";
    final int lastDot = mixinPackage.lastIndexOf('.');
    if (lastDot < 0) return "accessor";
    return mixinPackage.substring(0, lastDot) + ".accessor";
}
----

Example: `eu.lschreiber.testplugin.mixins` → `eu.lschreiber.testplugin.accessor`.

The accessor is placed in a _sibling_ `.accessor` package, not inside the mixin package.
SpongePowered Mixin does not allow classes in the mixin package itself to be loaded
directly by the application class loader; placing accessors outside the mixin package
avoids this restriction.

==== 10.3.5. Generated interface structure

For a mixin class targeting `MinecraftServer` with one `@GenerateAccessor` method, the
processor generates `MinecraftServerAccessor.java`:

[source,java]
----
package eu.lschreiber.testplugin.accessor;

import eu.lschreiber.laminateapi.accessor.LaminateAccessors;

/**
 * Generated by {@link eu.lschreiber.laminateapi.accessor.GenerateAccessorProcessor}.
 * Do not edit manually.
 */
public interface MinecraftServerAccessor {

    void write(String p0);           // one method per @GenerateAccessor

    static MinecraftServerAccessor of(Object target) {
        return LaminateAccessors.create(
            MinecraftServerAccessor.class, target,
            new String[]{"write", "testPlugin$write"});
    }
}
----

The `of(Object)` factory encodes a flat mapping array `[accessorName, mixinMethodName, ...]`
for `LaminateAccessors.create`.

[[laminate-accessors]]
=== 10.4. `LaminateAccessors`
// ============================================================

Class:: `eu.lschreiber.laminateapi.accessor.LaminateAccessors`

`LaminateAccessors` is the runtime half of the accessor system.
Generated `of(Object)` factories call `LaminateAccessors.create(...)`.

==== 10.4.1. `create`

[source,java]
----
public static <T> T create(
    Class<T> accessorInterface,
    Object target,
    String[] methodMappings    // flat [accessorName, mixinName, ...]
)
----

. Validates that `methodMappings.length` is even.
. Builds a `Map<String,String>` from the flat array (accessor name → mixin method name).
. Retrieves or creates a `ConcurrentHashMap<String,MethodHandle>` keyed by `target.getClass()`
  from a process-global `ConcurrentHashMap` cache.
. For each interface method: looks up the mixin method name, builds a descriptor key
  `"mixinName(paramType1,paramType2)"`, resolves and caches the `MethodHandle` if absent.
. Returns `Proxy.newProxyInstance(...)` with an `AccessorInvocationHandler`.

==== 10.4.2. `MethodHandle` resolution

[source,java]
----
private static MethodHandles.Lookup obtainLookup(Class<?> targetClass) {
    try {
        return MethodHandles.privateLookupIn(targetClass, MethodHandles.lookup());
    } catch (IllegalAccessException e) {
        return MethodHandles.lookup();
    }
}
----

Uses `MethodHandles.privateLookupIn` for wide access; falls back to the public lookup.
The resolved handle is cached to avoid repeated reflection on every call.

Resolution walks the full class hierarchy (`cls = cls.getSuperclass()` until `null`),
matching by name and erased parameter types.

==== 10.4.3. `AccessorInvocationHandler`

A private `record` implementing `InvocationHandler`.
Routes calls to `Object` methods (`toString`, `hashCode`, `equals`) to safe defaults,
and all other calls through the cached `MethodHandle`:

[source,java]
----
return handle.invokeWithArguments(prependTarget(target, args));
----

`prependTarget` prepends the mixin target object as the first argument, because unreflected
method handles for instance methods take the receiver as argument 0.

[[inject-event]]
=== 10.5. `@InjectEvent`, `At`, `Rewrite`, and `InjectEventProcessor`
// ============================================================

==== 10.5.1. Purpose

Plugin authors often want to fire a Bukkit event from inside an NMS method body.
The naive approach (writing a companion mixin by hand) is repetitive and error-prone.
`@InjectEvent` expresses the intent declaratively; the processor generates the companion
mixin automatically.

==== 10.5.2. `At` — injection point

[cols="1,3"]
|===
|Value |Meaning

|`HEAD`
|Before the first instruction in the target method.

|`TAIL`
|Just before every `RETURN` in the target method.

|`RETURN`
|Immediately before each `RETURN` opcode (same as `TAIL` in most cases).

|`INVOKE`
|Before a specific method call; requires `@InjectEvent#target()` to name the call site.
|===

Maps directly to SpongePowered Mixin's `@At` value.

==== 10.5.3. `@Rewrite` — type-safe method descriptor

[source,java]
----
@Target({})
@Retention(RetentionPolicy.CLASS)
public @interface Rewrite {
    String method();
    Class<?> returns() default void.class;
    Class<?>[] args() default {};
}
----

Used as the value of `InjectEvent.methodRewrite()` to construct an unambiguous JVM method
descriptor for overloaded NMS methods.
`InjectEventProcessor` converts a `@Rewrite` into a JVM descriptor string using
`JvmDescriptors.fromClass(...)`.

==== 10.5.4. `@InjectEvent` annotation

[source,java]
----
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.CLASS)
public @interface InjectEvent {
    String method() default "";           // simple name or full JVM descriptor
    Rewrite methodRewrite() default @Rewrite(method = "");  // type-safe alternative
    At at();
    Class<?> event();
    boolean cancellable() default false;
    String target() default "";           // for At.INVOKE: call-site descriptor
}
----

Exactly one of `method` and `methodRewrite` must be set; setting both or neither is a
compile-time error emitted by `InjectEventProcessor`.

`@Retention(CLASS)` — not needed at runtime; the annotation is consumed entirely at
compile time and only the generated companion mixin class remains.

==== 10.5.5. `InjectEventProcessor` — generated artefacts

For each `@InjectEvent`-annotated method the processor generates:

. A companion mixin class (e.g. `Generated_testPlugin$onTick_Mixin`) annotated with
  `@Mixin` targeting the same NMS class.
. Inside the companion: a `@Inject` method with the appropriate `@At` and target
  descriptor.
. The injected body calls `LaminateEventBridge.fire(...)` or
  `LaminateEventBridge.fireAndReturn(...)` (when `cancellable = true`) with the
  fully-qualified event class name, the NMS target object, and any extra arguments.

All generated companion classes for a plugin are collected into a single mixin JSON
configuration file named `mixins.<plugin>.generated.json`.

The plugin name is derived from the mixin class's package.
For `eu.lschreiber.testplugin.mixins.ServerLevelMixin`, the derived plugin name is
`testplugin`, producing `mixins.testplugin.generated.json`.

That file must be listed in `plugin.yml`:

[source,yaml]
----
mixins:
  - mixins.testplugin.json
  - mixins.testplugin.generated.json
----

==== 10.5.6. Cancellable injection — generated pattern

When `cancellable = true`, the generated `@Inject` method signature includes a
`CallbackInfo` parameter, and the generated body becomes:

[source,java]
----
Event event = LaminateEventBridge.fireAndReturn(
    "eu.lschreiber.testplugin.events.ServerLevelTickStartEvent",
    (Object) this,
    hasTime
);
if (event instanceof Cancellable c && c.isCancelled()) {
    ci.cancel();
}
----

[[event-bridge]]
=== 10.6. `LaminateEventBridge`
// ============================================================

Class:: `eu.lschreiber.laminateapi.event.LaminateEventBridge`

==== 10.6.1. The class-loader problem

Generated companion mixin classes are injected into NMS classes (e.g. `ServerLevel`).
NMS classes are loaded by the server's `AppClassLoader`.
Plugin event classes (e.g. `ServerLevelTickStartEvent`) are loaded by child
`PluginClassLoader` instances.
`AppClassLoader` cannot see classes defined in child class loaders.

A direct `new ServerLevelTickStartEvent(...)` in the generated bytecode would cause a
`NoClassDefFoundError` at runtime.

`LaminateEventBridge` solves this by being on `AppClassLoader`'s classpath (as part of
`laminate-api`, an `implementation` dependency of `paper-server`) while performing
dynamic class resolution through plugin class loaders at runtime.

==== 10.6.2. `fire` and `fireAndReturn`

[source,java]
----
public static void fire(String eventClassName, Object target, Object... extraArgs)

public static Event fireAndReturn(String eventClassName, Object target, Object... extraArgs)
----

Both delegate to `constructAndFire(...)` which:

. Calls `resolveConstructor(eventClassName, target, extraArgs)`.
. If the result is `UNRESOLVED_SENTINEL`: returns `null`.
. Prepends `target` to `extraArgs` to build the constructor argument array.
. Instantiates the event class with `ctor.newInstance(ctorArgs)`.
. Calls `Bukkit.getPluginManager().callEvent(event)`.
. Returns the event (or `null` on `Exception`).

==== 10.6.3. Constructor resolution and caching

Cache key: `"<eventFQN>|<targetClass>,<argType1>,<argType2>,..."`

On cache miss:

. Attempts to load the event class from every registered plugin's class loader via
  `plugin.getClass().getClassLoader().loadClass(eventClassName)`.
. If no plugin can load it: logs a warning, stores `UNRESOLVED_SENTINEL`, returns.
. Iterates `eventClass.getDeclaredConstructors()` to find one whose parameter types are
  compatible with the runtime argument types.
. Compatibility is checked per-parameter: `null` matches any non-primitive; boxed types
  match their primitives; standard `isAssignableFrom` applies for the rest.
. Calls `ctor.setAccessible(true)` and stores the result with `putIfAbsent`.

`UNRESOLVED_SENTINEL` is `Object.class.getDeclaredConstructor()` — a `Constructor<Object>`
instance used purely as a sentinel to avoid repeated failed lookups.

[[internal-utils]]
=== 10.7. Internal utilities
// ============================================================

==== 10.7.1. `MixinTargetResolver`

Class:: `eu.lschreiber.laminateapi.internal.MixinTargetResolver`

Shared by both `GenerateAccessorProcessor` and `InjectEventProcessor`.
Resolves the single target class declared in a `@Mixin` annotation on a given
`TypeElement`.

Supports two forms:

* `@Mixin(SomeClass.class)` — reads the `value` attribute as `List<AnnotationValue>`,
  extracts the first `TypeMirror`, resolves it to a `TypeElement`.
* `@Mixin(targets = "com.example.SomeClass")` — reads the `targets` attribute as
  `List<AnnotationValue>`, extracts the first string, splits on the last `.` to get
  simple name.

Emits a compile-time error if:

* More than one target is declared (multiple-target mixins are not supported by either
  processor).
* No `@Mixin` annotation is found on the enclosing class.

Returns a `MixinTarget(String simpleName, String fqn)` record, or `null` on failure.

==== 10.7.2. `JvmDescriptors`

Class:: `eu.lschreiber.laminateapi.internal.JvmDescriptors`

Converts Java types to JVM type descriptor strings.

[source,java]
----
public static String fromClass(Class<?> type)         // runtime Class
public static String fromTypeMirror(TypeMirror mirror) // compile-time TypeMirror
----

Primitive mapping:

[cols="1,1"]
|===
|Java type |JVM descriptor

|`void` |`V`
|`boolean` |`Z`
|`byte` |`B`
|`char` |`C`
|`short` |`S`
|`int` |`I`
|`long` |`J`
|`float` |`F`
|`double` |`D`
|array |`[` + component descriptor
|reference |`L` + binary name with `/` separators + `;`
|===

`fromTypeMirror` handles `TypeElement` nesting correctly: inner classes use `$` as the
separator in binary names (`Map.Entry` → `"java.util.Map$Entry"`).
